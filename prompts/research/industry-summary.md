# Industry Summary Prompts

用于 **Nexus Research：AI行业研究与情报分析平台**。

## 文件说明

本文件沉淀行业信息总结类 Prompt，主要用于将行业新闻、研报片段、公司公告等非结构化文本，转化为结构化研究笔记。

本文件包含 3 类 Prompt：

- Prompt 1：基础行业信息总结
- Prompt 2：带分类判断的行业信息总结
- Prompt 3：Few-shot 行业信息总结

---

## Prompt 1：基础行业信息总结

### 适用场景

适用于对一段行业新闻、研报片段、公司公告进行结构化总结。

### 设计重点

- 明确模型角色：AI 行业研究分析师
- 限制模型不要编造信息
- 要求输出结构化研究笔记
- 提取主体、事件、影响和不确定信息

### Prompt 内容

```text
你是一个 AI 行业研究分析师。

你的任务是对用户提供的行业信息进行结构化总结，帮助读者快速理解核心内容。

请阅读以下文本：

"""
{input_text}
"""

请遵守以下要求：
1. 不要编造原文没有的信息。
2. 优先提炼与行业趋势、公司动态、技术变化、市场影响相关的信息。
3. 如果原文信息不足，请明确说明“原文未提及”。
4. 输出内容要简洁、专业、适合用于研究笔记。

请按照以下格式输出：

## 一句话总结

## 关键信息
- 
- 
- 

## 涉及主体
- 公司/机构：
- 产品/技术：
- 人物：

## 可能影响
- 对行业：
- 对公司：
- 对用户/客户：

## 不确定信息
- 
```

### 输入示例

```text
某 AI 公司发布了新一代企业级智能体平台，支持多工具调用、权限管理和工作流编排，主要面向金融、制造和政企客户。
```

### 预期输出

输出应包含以下部分：

- 一句话总结
- 关键信息
- 涉及主体
- 可能影响
- 不确定信息

## Prompt 2：带分类判断的行业信息总结

### 适用场景

适用于将行业信息写入研究数据库，需要同时完成总结、分类和影响判断的场景。

### 设计重点

- 限定分类标签，减少输出发散
- 使用 JSON 输出，便于后续程序解析
- 明确要求只基于原文，不补充外部信息
- 将总结结果转化为可结构化存储的数据

### Prompt 内容

```text
你是一个 AI 行业研究助理。

请分析用户提供的行业文本，并完成“总结 + 分类 + 影响判断”。

文本如下：

"""
{input_text}
"""

分类标签只能从以下选项中选择：
- 技术突破
- 产品发布
- 融资并购
- 政策监管
- 市场竞争
- 商业合作
- 其他

要求：
1. 只基于原文分析，不要补充外部信息。
2. 如果无法判断，请选择“其他”，并说明原因。
3. 输出要适合写入行业研究数据库。
4. 请只输出原始 JSON 对象，不要使用 Markdown 代码块，不要添加解释文字。
5. key_points 控制在 3-5 条，每条不超过 40 个中文字符。
6. uncertainties 只记录原文未提及、无法确认或需要进一步验证的信息。

输出格式：

{
  "summary": "一句话总结",
  "category": "分类标签",
  "key_points": ["要点1", "要点2", "要点3"],
  "entities": {
    "companies": [],
    "products": [],
    "technologies": []
  },
  "impact": {
    "industry": "对行业的影响",
    "company": "对相关公司的影响",
    "users": "对用户或客户的影响"
  },
  "uncertainties": ["不确定信息1"]
}
```

### 输入示例

```text
一家大模型创业公司宣布完成 B 轮融资，并表示资金将主要用于多模态模型研发和企业级产品商业化。
```

### 预期输出

输出应为合法 JSON，包含以下字段：

- `summary`
- `category`
- `key_points`
- `entities`
- `impact`
- `uncertainties`

## Prompt 3：Few-shot 行业信息总结

### 适用场景

适用于希望模型模仿固定风格，输出统一格式研究摘要的场景。

### 设计重点

- 通过示例约束输出风格
- 让模型学习“简洁、研究化、结构化”的表达方式
- 适合后续用于自动生成统一风格的研究笔记
- 降低不同输入文本导致的输出风格差异

### Prompt 内容

```text
你是一个专业的 AI 行业研究分析师。

请参考示例的风格，对输入文本进行总结。
请避免使用“第一”“最强”“最快”等绝对化表述，除非原文明确使用该表述。

示例 1：

输入：
OpenAI 发布了新的企业级功能，支持团队统一管理模型调用权限和数据访问策略。

输出：
## 一句话总结
OpenAI 推出企业级管理能力，强化组织级模型使用和数据治理。

## 关键信息
- OpenAI 发布新的企业级功能
- 功能重点包括权限管理和数据访问策略
- 目标用户是团队和企业组织

## 可能影响
- 提升企业采用 AI 工具的安全性
- 降低组织内部模型使用的管理成本

示例 2：

输入：
某云厂商上线新的 AI Agent 开发平台，提供模型接入、工具调用、插件市场和企业权限管理能力。

输出：
## 一句话总结
某云厂商推出 AI Agent 开发平台，强化企业级智能体构建与管理能力。

## 关键信息
- 平台支持模型接入、工具调用和插件市场
- 提供企业权限管理能力
- 目标是降低企业构建 AI Agent 的门槛

## 可能影响
- 推动企业级 Agent 应用落地
- 加剧云厂商在 AI 开发平台领域的竞争
- 为开发者提供更完整的 Agent 工程化工具链

现在请处理以下文本：

"""
{input_text}
"""

请按照示例格式输出。
```

### 输入示例

```text
一家企业软件公司推出面向销售团队的 AI 助手，支持自动总结客户沟通记录、生成跟进建议，并与 CRM 系统集成。
```

### 预期输出

输出应模仿示例结构，包括：

- 一句话总结
- 关键信息
- 可能影响

---

## 使用建议

### 使用顺序

建议按照以下顺序练习：

1. 先使用 Prompt 1，理解基础结构化总结。
2. 再使用 Prompt 2，练习 JSON 格式输出和分类判断。
3. 最后使用 Prompt 3，体会 Few-shot 示例对输出风格的约束作用。

### 适配 Nexus Research 的方向

后续在 Nexus Research 项目中，可以基于本文件扩展以下能力：

- 行业新闻摘要
- 公司动态分析
- 技术趋势识别
- 政策监管信息分类
- 竞品情报整理
- 研究报告自动生成

### 后续可优化方向

- 增加可信度评分字段
- 增加信息来源字段
- 增加时间、地区、行业标签
- 增加适合 RAG 检索的元数据结构
- 将 JSON 输出与 Pydantic Schema 对齐
